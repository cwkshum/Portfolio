SeeRem

Front-End Website

Overview

For this project, I worked in a team of two with the SEEDA company to develop a front-end website for their SeeRem division. The objective of the project was to design and build a functioning website for SeeRem to allow their customers to learn more about the SeeRem application and to connect with the sales team. By communicating with the company, I was able to learn more about the service they were offering to help my partner and I better design for the intended purpose of the website. The end product was a fully developed front-end responsive website to promote SeeRem, an application that allows teams working in remote worksites to connect with one another.

Ideation & Style Guide

During the first month of the project, I began creating wireframes and a style guide for the SeeRem website. The style guide consisted of designing the basic structure and elements that would be incorporated into the site which would help my partner and I keep a consistent design and code structure on all the web pages. This stage also included understanding the branding of the company and how it would be represented through the website.

Website Development

Development of the website began in the second month, where I started creating template pages using HTML and CSS. Throughout the weeks, we began refining the layout of the overall site by compiling content, images, and deciding on the overall flow that would create a positive user experience. JavaScript was also used in the site to increase the functionality, such as having a scroll to top and confirmation messages.

Challenge of Responsiveness

Once the structure of the website was developed, I had to ensure that the site was responsive so that it can be viewed on different device screen sizes, including mobile. This was one of the more challenging tasks of the project as it was my first time developing a website that would be fully responsive. Making the website work well for mobile devices proved to be very hard as the screen size was a lot smaler in comparison to a desktop screen size. However, through the use of media queries, I was able to make the elements of the page more appropriate for smaller screen sizes.
